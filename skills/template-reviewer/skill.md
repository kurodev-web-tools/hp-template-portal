# Template Reviewer Skill

## 概要
ビジネスHTML テンプレートの包括的レビューを実施。デザイン一貫性、コード品質、SEO、アクセシビリティ、モバイル対応、パフォーマンスを評価し、改善推奨事項を出力します。

## トリガー
- 「テンプレートをレビューして」
- 「HTML の品質チェックをして」
- 「コードのレビューをしてください」
- テンプレートディレクトリが指定された場合

## レビュー手順

### Step 1: ファイル構成の把握
1. 対象ディレクトリ内の全 HTML ファイルを特定
2. CSS ファイル（Tailwind input / 独自 CSS）を特定
3. JS ファイル（スクリプト）を特定

### Step 2: 各ページの精査
各 HTML ファイルについて以下を確認：
- メタタグ（title, description, viewport, OG タグ）
- 外部リソース依存（画像、フォント、スクリプト）
- アクセシビリティ実装（ARIA 属性、キーボード操作）
- レスポンシブ対応
- ダークモード対応

### Step 3: デザインシステム分析
- CSS 設計の統一性（Tailwind / 独自 CSS の混在チェック）
- **独自 CSS のみの場合も同様に分析**
  - CSS ファイルの構造（モジュール性、命名規則）
  - 変数定義（CSS カスタムプロパティ、Sass 変数）
  - 一貫性のあるクラス命名
- カスタムクラスの一貫性
- カラーパレット、フォント設定

### Step 4: 問題点の抽出
以下の観点で問題点を分類：
- **重要度：高** - 機能障害、重大な SEO 問題
- **重要度：中** - メタ記述欠落、外部依存リスク
- **重要度：低** - アクセシビリティ改善、パフォーマンス

## チェックリスト

### メタタグ
- [ ] title タグの存在
- [ ] meta description（40-50 字）
- [ ] viewport 設定
- [ ] OG タグ（og:title, og:description, og:image）
- [ ] Twitter カード

### デザイン一貫性
- [ ] 全ページで同じ CSS 設計
- [ ] カラーパレットの統一
- [ ] フォント設定の統一
- [ ] 間隔・余白の統一

### アクセシビリティ
- [ ] ARIA ラベル
- [ ] aria-expanded / aria-controls
- [ ] キーボード操作（Tab, Enter, Escape）
- [ ] フォーカスインジケータ
- [ ] コントラスト比

### モバイル対応
- [ ] レスポンシブデザイン
- [ ] モバイルメニュー
- [ ] タッチターゲットサイズ（44px 以上）
- [ ] スクロールロック（メニュー開閉時）

### パフォーマンス
- [ ] 外部画像の最小化
- [ ] 遅延読み込み（loading="lazy"）
- [ ] CSS/JS の最小化
- [ ] WebP フォーマット使用

### SEO
- [ ] 適切な見出し構造（h1 → h2 → h3）
- [ ] 内部リンクの整合性
- [ ] 画像の alt 属性
- [ ] canonical タグ

## 外部依存カウント方法

1. **外部画像**: `src="http` または `src="//"` をカウント
2. **外部フォント**: Google Fonts, Adobe Fonts などの URL
3. **外部スクリプト**: CDN 由来の JS ファイル

## 評価基準

### 星評価（5 段階）

| 星 | 意味 |
|----|------|
| ★★★★★ | 完璧 - 改善点ほぼなし |
| ★★★★☆ | 優秀 - 軽微な問題あり |
| ★★★☆☆ | 普通 - 中程度の問題あり |
| ★★☆☆☆ | 要改善 - 重大な問題あり |
| ★☆☆☆☆ | 不適 - 全面的な修正が必要 |

## 注意事項

1. 各問題点には必ず**該当ページ**と**重要度**を明記
2. コードスニペットは具体的に示す
3. 推奨事項には**工数見積もり**を添える
4. 比較テーブルは既存レビュー結果と整合性を保つ
5. 総合評価は根拠とともに説明する
