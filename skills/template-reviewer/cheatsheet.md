# Template Reviewer - チートシート

## レビュー対象ディレクトリの指定方法

```
/public/templates/business/[a-z]/
```

## 確認すべきファイル

### 必須
- [ ] `index.html` - ヒーロー/ブランド紹介
- [ ] `about.html` - 思想/理念
- [ ] `service.html` - サービス/機能
- [ ] `contact.html` - お問い合わせフォーム
- [ ] `assets/css/tailwind-input.css` - Tailwind 設定
- [ ] `assets/js/*.js` - スクリプト類

### オプション
- [ ] 追加ページ（strategy.html, performance.html など）
- [ ] `assets/css/style.css` - 独自 CSS

---

## クイックチェックリスト

### 1. メタタグ（各ページ）
```bash
# grep で一括チェック
grep -r "meta name=\"description\"" public/templates/business/z/
grep -r "og:" public/templates/business/z/
```

### 2. 外部依存
```bash
# 外部画像
grep -r "src=\"http" public/templates/business/z/

# 外部フォント
grep -r "fonts.googleapis.com" public/templates/business/z/

# 外部スクリプト
grep -r "src=\"http" public/templates/business/z/ | grep "\.js"
```

### 3. アクセシビリティ
```bash
# ARIA 属性
grep -r "aria-" public/templates/business/z/

# キーボード操作
grep -r "Escape" public/templates/business/z/
grep -r "keydown" public/templates/business/z/
```

### 4. デザイン統一性
```bash
# CSS ファイル確認
ls -la public/templates/business/z/assets/css/

# Tailwind 使用状況
grep -r "@import.*tailwindcss" public/templates/business/z/

# 独自 CSS の場合
grep -r "\.css" public/templates/business/z/
```

### 5. AI デザインっぽさチェック
```bash
# 情報設計
grep -r "hero\|main\|featured" public/templates/business/z/
grep -r "cta\|action\|button" public/templates/business/z/

# 色設計
grep -r "color\|background" public/templates/business/z/assets/css/

# 装飾過多チェック
grep -c "class=" public/templates/business/z/*.html
```

## 評価マトリックス

| 項目 | 5 点 | 3 点 | 1 点 |
|------|------|------|------|
| **デザイン一貫性** | 全ページ統一 | 一部統一 | バラバラ |
| **コード品質** | 整理された構造 | 標準的 | 要リファクタ |
| **アクセシビリティ** | 完全実装 | 一部実装 | 未実装 |
| **SEO** | 完璧 | 一部不足 | 重大な問題 |
| **モバイル** | 完全対応 | 一部対応 | 未対応 |
| **パフォーマンス** | 最適化済 | 標準 | 重たい |

---

## 問題点の重要度分類

### 高（High）
- 機能しない機能
- 重大な SEO 問題（title/description 欠落）
- アクセシビリティの重大な欠陥
- セキュリティリスク

### 中（Medium）
- メタ記述の欠落（サブページ）
- OG タグ未実装
- 外部画像依存（リンク切れリスク）
- デザインシステムの軽微な不統一

### 低（Low）
- パフォーマンス改善の余地
- コードの可読性向上
- オプション機能の欠如

## AI デザインっぽさ チェックリスト

### 情報設計・優先順位（5 つの視点）

| # | チェック項目 | 確認ポイント |
|---|-------------|-------------|
| 1 | 情報の優先順位 | 整っているだけの平均点デザインになっていないか |
| 2 | 画面の主役 | 画面内で何が主役なのかがひと目でわかるか |
| 3 | 視線の流れ | 「最初に見る → 次に比較する → 最後に行動する」という流れがあるか |
| 4 | UI パータンの適切さ | よくある UI パターンを機械的に並べていないか |
| 5 | 業務・体験への適合 | この業務・体験に合った情報設計から考えられているか |

### 色設計（3 つの視点）

| # | チェック項目 | 確認ポイント |
|---|-------------|-------------|
| 1 | 色の役割 | ブランド色・操作色・状態色・注意色など役割が明確か |
| 2 | 色の一意性 | 同じ色に複数の意味を持たせていないか |
| 3 | 色の一貫性 | 色の使い方に一貫性があるか |

### 実運用のリアリティ（3 つの視点）

| # | チェック項目 | 確認ポイント |
|---|-------------|-------------|
| 1 | データの耐性 | 長いテキスト、欠損、異常値、ゼロ件、重複などがあっても破綻しにくい UI か |
| 2 | 現実的な状態 | 理想的なモックではなく、実際に使われるプロダクトとして自然な UI か |
| 3 | 現実データ想定 | 現実的なデータ状態を想定した設計になっているか |

### 削ぎ落とし・ミニマリズム（3 つの視点）

| # | チェック項目 | 確認ポイント |
|---|-------------|-------------|
| 1 | 要素の絞り込み | 情報や装飾を盛りすぎていないか |
| 2 | 必要最低限 | 本当に必要な要素に絞られているか |
| 3 | 完成度 | 最後に削ぎ落としたような完成度になっているか |

### 総合評価

**AI デザインっぽさ：低 / 中 / 高**

| レベル | 特徴 |
|--------|------|
| **低** | 情報設計が明確、色が役割を持つ、実運用を想定、削ぎ落とされている |
| **中** | 一部机械的、色の役割が曖昧、理想データ前提、装飾過多の傾向 |
| **高** | 平均点デザイン、主役不明、視線の流れなし、色は見た目優先、盛りすぎ |
